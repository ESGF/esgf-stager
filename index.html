Stager for ESGF

The Stager provides the means to transparently access files located at a remote system. Remote files are cached locally preserving the directory structure from the remote structure.

Implementation

The current implementation uses a Java web Filter for intercepting the file requests targeted at servlets running locally. This calls are handled differently depending on the user-agent attribute from the http request header.
If the request was started from within a browser, the user is redirected to a "waiting page" from where it will be automatically redirected to the original servlet when the file is ready.
In case the request was started by a tool, the request will just block until the file is available
The file retrieval will be triggered only once and in cache the file is already in the cache the filter will return immediately allowing the call to proceed normally without any delay. This call is logged to optimize the cache-cleaning mechanism.
